# Чеклист для ручного тестирования MODOCRM

## Подготовка
- Приложение запущено на `http://localhost:3000`
- Тестовые пользователи созданы:
  - `user@modocrm.com` / `User123456`
  - `admin@modocrm.com` / `Admin123456`

## 1. Тестирование авторизации

### 1.1 Вход в систему
- [ ] Открыть `http://localhost:3000/auth/signin`
- [ ] Ввести `user@modocrm.com` / `User123456`
- [ ] Нажать "Войти"
- [ ] Проверить редирект на `/clients`
- [ ] Проверить, что имя пользователя отображается в интерфейсе

### 1.2 Выход из системы
- [ ] Найти кнопку меню пользователя
- [ ] Нажать "Выйти"
- [ ] Проверить редирект на `/auth/signin`

### 1.3 Защита маршрутов
- [ ] Выйти из системы
- [ ] Попытаться открыть `http://localhost:3000/clients`
- [ ] Проверить редирект на `/auth/signin`
- [ ] Повторить для `/proposals`, `/suppliers`, `/settings`

## 2. Тестирование CRUD операций для клиентов

### 2.1 Создание клиента (физическое лицо)
- [ ] Перейти на `/clients`
- [ ] Нажать кнопку "Создать" или "Новый"
- [ ] Выбрать тип "Физическое лицо"
- [ ] Заполнить форму:
  - Имя: "Тест"
  - Фамилия: "Клиент"
  - Телефон: "+39 333 123 4567"
  - Email: "test.client@example.com"
- [ ] Нажать "Сохранить"
- [ ] Проверить, что клиент появился в списке

### 2.2 Создание клиента (юридическое лицо)
- [ ] Нажать "Создать"
- [ ] Выбрать тип "Юридическое лицо"
- [ ] Заполнить форму:
  - Название компании: "Тестовая Компания"
  - VAT: "IT12345678901"
  - Телефон: "+39 333 123 4568"
  - Email: "company@example.com"
- [ ] Сохранить и проверить

### 2.3 Просмотр списка клиентов
- [ ] Проверить, что список клиентов отображается
- [ ] Проверить поиск по имени/email
- [ ] Проверить фильтры (если есть)

### 2.4 Редактирование клиента
- [ ] Найти созданного клиента в списке
- [ ] Нажать кнопку редактирования
- [ ] Изменить имя на "Обновленный"
- [ ] Сохранить
- [ ] Проверить, что изменения отобразились

### 2.5 Удаление клиента
- [ ] Создать тестового клиента для удаления
- [ ] Нажать кнопку удаления
- [ ] Подтвердить удаление
- [ ] Проверить, что клиент удален из списка

## 3. Тестирование CRUD операций для поставщиков

### 3.1 Создание поставщика
- [ ] Перейти на `/suppliers`
- [ ] Нажать "Создать"
- [ ] Заполнить форму:
  - Название: "Тестовый Поставщик"
  - Телефон: "+39 333 111 2222"
  - Email: "supplier@test.com"
- [ ] Сохранить и проверить

### 3.2 Редактирование и удаление поставщика
- [ ] Отредактировать поставщика
- [ ] Удалить тестового поставщика (если нет связанных данных)

## 5. Тестирование создания документов (предложений)

### 5.1 Создание предложения
- [ ] Перейти на `/proposals`
- [ ] Нажать "Создать предложение" или "Новый"
- [ ] Выбрать клиента из списка

### 5.2 Использование конфигуратора продуктов
- [ ] Добавить группу продуктов
- [ ] Выбрать категорию продукта (например, "Porte interne")
- [ ] Выбрать поставщика
- [ ] Добавить позицию:
  - Выбрать продукт из списка
  - Указать количество: 2
  - Указать цену: 150.00
  - Выбрать параметры (если есть)
- [ ] Проверить HEX → RAL конвертацию цветов (если применимо)
- [ ] Сохранить предложение

### 5.3 Просмотр и редактирование предложения
- [ ] Просмотреть созданное предложение
- [ ] Проверить генерацию PDF (если доступно)
- [ ] Отредактировать предложение
- [ ] Изменить статус предложения
- [ ] Удалить тестовое предложение
